# Chess Openings Trainer

## Workflow

1. Read `steps.md` — find the first unchecked step (`- [ ]`)
2. Implement that step
3. Ask the user if the step can be marked complete
4. If YES: check it off (`- [x]`) in `steps.md`, then move to the next unchecked step
5. If NO: address feedback, then ask again

## Reference

- `plan.md` has full spec, data model, UI layout, and styling notes
- Tech: React + TS + Vite, react-chessboard, chess.js, Zustand, Tailwind, PWA
- Chess.com dark theme: bg #312e2b, board #779952/#eeeed2

## Style

- Be extremely concise, sacrifice grammar for concision
- Commit messages: short, no fluff
